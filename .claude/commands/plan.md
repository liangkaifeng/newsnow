---
description: 规划新功能的实施方案（创建 Spec 文档）
---

请按照以下步骤规划新功能：

1. **需求确认**
   - 询问我功能的详细需求
   - 确认用户场景和目标
   - 明确范围（做什么、不做什么）

2. **技术方案设计**
   创建 `.specs/[feature-name].md` 文档，包含：
   - 需求背景和用户故事
   - 功能范围（In Scope / Out of Scope）
   - 技术架构设计（架构图、数据流）
   - 数据库设计（SQL Schema）
   - API 设计（端点、请求/响应格式）
   - 前端组件设计
   - E2E 测试计划
   - 实施计划（任务分解、工时估算）
   - 成本估算（开发成本 + 运营成本）
   - 风险管理

3. **更新总体计划**
   更新 `.specs/master-implementation-plan.md`：
   - 添加到对应阶段
   - 更新时间估算
   - 更新总成本

4. **创建 E2E 测试框架**
   在 `e2e/` 目录创建测试文件，编写测试用例（先标记为 skip）

5. **确认开始实施**
   询问我是否立即开始实施，或等待审查

**重要**：
- 所有文档使用中文
- 技术方案要简单、高效、低成本
- 必须包含成本估算
- 优先考虑开源方案
