description: 实施 Spec 文档中规划的功能
---

请按照以下流程实施功能：

1. **确认 Spec 文档**
   - 检查 `.specs/` 目录中对应的规格文档
   - 如果没有 spec 文档，先执行 `/plan` 命令

2. **创建 Todo List**
   使用 TodoWrite 工具创建任务清单，包括：
   - 数据库设计（如需要）
   - API 端点实现
   - 前端组件开发
   - E2E 测试编写
   - 集成测试
   - 文档更新

3. **逐步实施**
   按照 spec 文档的实施计划，依次完成：
   - Day 1: 后端基础（数据库、API）
   - Day 2: 认证/核心逻辑
   - Day 3: 前端组件
   - Day 4: UI/UX 优化
   - Day 5: 测试和部署

4. **E2E 测试优先**
   - 每完成一个模块，立即编写/启用对应的 E2E 测试
   - 测试必须通过才能继续下一步
   - 运行 `pnpm test:e2e` 验证

5. **代码审查**
   自我审查清单：
   - [ ] 代码符合 ESLint 规则
   - [ ] TypeScript 类型完整
   - [ ] 复杂逻辑有注释
   - [ ] 性能优化（缓存、懒加载）
   - [ ] 安全检查（XSS、SQL 注入、CSRF）
   - [ ] 错误处理完善

6. **提交代码**
   - 编写清晰的中文提交信息
   - 格式：`feat: 添加 xxx 功能`
   - 提交前运行 `pnpm test:e2e`
   - 推送到远程分支

7. **验收确认**
   - 所有 E2E 测试通过 ✅
   - Lighthouse Performance > 90 ✅
   - 成本在预算内 ✅
   - 用户体验流畅 ✅

**注意**：
- 遵循 Spec Kit 流程
- E2E 测试必须通过
- 主动报告进度
- 遇到问题及时沟通
